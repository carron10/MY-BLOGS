Understanding version control

Version Control Systems(VCS) are the software tools for tracking/managing all the changes made to the source code during the project development. It stores your project(s) and allows you to view all the changes that you have made. What does this mean?, Think when you are developing your projects, their are many changes that you are making, where you are fixing bugs,deleting and updating codes. Sometimes you delete some codes and wish if you can get them back and that is where VCS comes in. A Version Control System allows you to view those codes that you have deleted by letting you see all changes that you have made from the starting point.

Types of Version Control System
  1. Local Version Control System
  2. Centralized Version Control System
  3. Distributed Version Control System
Local Version Control System

Local Version Control is the one which is located on your computer. To use a Local Version Control you need to download and install it on your pc. The advantage of Local version control is that you don't always need internet in order for you to view the changes that you have made. The great disadvantage is that if your machine crashes all your project on Local Version Control will be erased

Centralized Version Control System

Centralized Version Control System is when the projects are put in one server that is accessible to your team. Think about it when your are developing one project with your friends, the good thing is to put the project in a place where your friends can edit it with their own machines.In the Centralized Version Control Systems, there will be a single central server that contains all the files related to the project

You can see that the only drawback of using centralized Version Cotrol is that, if it crashes everything will lost. Sorry if you have been wondering where the word Repository comes in,well this is just a place to store your project e.i I mean your project(s) in Version Control System

Distributed Version Control System

To overcome the problems of the above two, we have Distributed Version Control System which combines those above and make sure that there is also a VCS on your pc and the other(S) accessible to everyone. In a distributed version control system, there will be one or more servers and many collaborators similar to the centralized system. In this case if you have a project that you are doing with others, all the collaborators of that project will have their repository with the exact copy of project that you have.

Difference between Git and GitHub

We have introduced each other to Version Conrol Systems, they is no doubt that these two are examples of Version Control Systems. Yes, Git and Github they are Version Control Systems but they work differently and they are originated from different Companies.

Git is a Local Version Control System which tracks changes of the source codes on your PC. To use Git you need to download and install it to your machine. It can be installed on the most common operating systems like Windows, Mac, and Linux. On the other hand Github is a Centralized Version Control System. It is used to host your source codes through web cloud services. The best thing about github is that even when your PC is at home you can go to your project repository and view/edit the codes. For you to use Github you need to create an account at https://github.com/. Github works with Git whereby you push/update your source codes from your local repository to your github repository(remote repository). However using Git doesn't depend on github.
